# Buyursin Technics — Building Automation Website

Полноценный сайт инженерного интегратора Buyursin Technics в светлом Apple-inspired стиле: строгая сетка, статичные понятные схемы, минимум декоративной анимации и фокус на бизнес-выгоде.

## Основные возможности

- Равноправная коммуникация для четырёх аудиторий: владелец, управляющая компания, девелопер, технический директор.
- Русская и узбекская локализация.
- Пять понятных сценариев: энергия, климат, протечка, доступ, пожар.
- Интерактивный предварительный расчёт эффективности по средним диапазонам.
- Отдельные страницы решений, проектов, компании, контактов и калькулятора.
- Рабочие формы заявок с серверной доставкой через webhook или Telegram Bot и безопасным email fallback.
- Реальные контакты из презентации: телефон, email и QR-ссылка.
- Внутренний защищённый конструктор КП по адресу `/admin/proposal`.
- Экспорт КП в PDF и полностью редактируемый PPTX, горизонтальные слайды 16:9.
- Пять слайдов КП: компания, предложение, решения, предварительная стоимость, контакты и CTA.
- Русская и узбекская версии, динамические цены, тексты, логотип клиента и фотографии объекта.
- Next.js 16, React 19, Tailwind CSS 4, TypeScript.
- Node.js 24 для Vercel.

## Маршруты

## Серверные переменные

Создайте `.env.local` на основе `.env.example`.

### Доставка заявок

Можно настроить один или оба варианта:

```text
LEAD_WEBHOOK_URL
LEAD_WEBHOOK_TOKEN
TELEGRAM_BOT_TOKEN
TELEGRAM_CHAT_ID
```

Если доставка не настроена, форма безопасно открывает почтовое приложение с заполненной заявкой на `info@btechnics.uz`.

### Внутренний конструктор КП

```text
PROPOSAL_ACCESS_CODE
PROPOSAL_SESSION_SECRET
```

Обе переменные обязательны. Они используются только на сервере и не должны иметь префикс `NEXT_PUBLIC_`.

Кнопка **«Сформировать КП»** находится в подвале сайта. После входа сотрудник может
выбрать язык, услуги, валюту, цены, расчётные параметры, загрузить логотип и фото,
а затем скачать `.pptx`. Тексты, числа и изображения в готовом PowerPoint не
сведены в один скриншот и могут редактироваться по отдельности.

Для генерации секрета:

```bash
openssl rand -hex 32
```

## Vercel

Самый простой путь — подключить `RawenHold/Buyursin` через Vercel Git Integration. Vercel автоматически распознает Next.js и Node.js 24 из `package.json`.

В настройках проекта добавьте серверные переменные из раздела выше. Не задавайте вручную `npm ci` в Install Command — оставьте Auto.

Дополнительно в `.github/workflows` подготовлены проверки и деплой через Vercel CLI при наличии secrets:

```text
VERCEL_TOKEN
VERCEL_ORG_ID
VERCEL_PROJECT_ID
```

## Безопасность

- API-токены и коды доступа отсутствуют в клиентском коде.
- Сессия конструктора КП хранится в `HttpOnly` cookie.
- Формы валидируются серверным route handler.
- `.env*`, `.vercel`, `.next` и `node_modules` исключены из Git.
- Перед коммитом выполняется автоматический secret scan.

## Контент и цифры

Публично используются только данные, указанные в презентации компании:

- потенциальная экономия энергозатрат до 30%;
- ориентир окупаемости 1–3 года;
- 15 лет опыта;
- список объектов, брендов, партнёров и сертификаций.

Калькулятор всегда отмечает результат как предварительную оценку, требующую технического аудита.
